### Weekly Outcomes

* Gained a solid understanding of AWS Storage services and their use cases.

* Learned how to use **Amazon S3** to:

  * Create and manage S3 buckets.
  * Store and retrieve objects.
  * Configure bucket policies and IAM permissions.
  * Understand Amazon S3 Storage Classes for cost optimization.

* Completed hands-on labs with **AWS Backup**, including:

  * Creating Backup Vaults.
  * Configuring Backup Plans.
  * Managing Recovery Points.
  * Restoring AWS resources from backups.

* Explored **AWS Import/Export** concepts and data migration strategies for transferring large datasets to AWS.

* Completed practical exercises with **AWS Storage Gateway**, including:

  * Understanding hybrid cloud storage architectures.
  * Integrating on-premises storage with AWS.
  * Exploring File Gateway, Volume Gateway, and Tape Gateway use cases.

* Gained hands-on experience with **Amazon FSx**, including:

  * Creating managed file systems.
  * Understanding Amazon FSx deployment options.
  * Providing high-performance shared file storage for enterprise workloads.

* Developed an understanding of:

  * Data protection strategies.
  * Backup and disaster recovery concepts.
  * High availability and data durability.
  * Storage lifecycle management and cost optimization.

### Knowledge Gained

* Understood the architecture and core concepts of Amazon S3 object storage.
* Learned how to select appropriate Amazon S3 Storage Classes based on access patterns and cost requirements.
* Understood backup and restore workflows using AWS Backup.
* Gained knowledge of hybrid cloud storage using AWS Storage Gateway.
* Learned the use cases and deployment models of Amazon FSx.
* Explored AWS solutions for large-scale data migration.
* Improved understanding of designing resilient storage architectures that support business continuity and disaster recovery.

### Challenges

* Encountered networking challenges while configuring connectivity between an Amazon EC2 instance and AWS Storage Gateway.
* Troubleshot VPC networking components, including route tables and security groups, to establish successful communication.

### Objectives for Next Week

* Learn the **AWS Shared Responsibility Model**.
* Understand the security responsibilities shared between AWS and customers.
* Explore AWS Identity and Access Management (IAM) and AWS security best practices.
