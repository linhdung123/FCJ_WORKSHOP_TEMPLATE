### Week 5 Achievements:

* Gained an understanding of the **AWS Shared Responsibility Model** and how AWS security services contribute to protecting cloud infrastructure and data.

* Improved practical skills with **AWS Identity and Access Management (IAM)** by:
  * Creating and managing IAM Users, Groups, and Roles.
  * Designing IAM Policies based on the **Principle of Least Privilege**.
  * Applying **IAM Conditions** to enforce fine-grained access control.
  * Configuring IAM Roles for applications to securely access AWS resources without storing long-term credentials.

* Learned how to use **AWS Resource Tags** to:
  * Organize AWS resources.
  * Improve resource management.
  * Implement **Attribute-Based Access Control (ABAC)** using resource tags.

* Completed hands-on labs with **AWS Security Hub**, gaining experience in:
  * Aggregating security findings from multiple AWS services.
  * Monitoring the overall security posture of AWS accounts.
  * Identifying security misconfigurations and compliance issues.

* Learned how **AWS IAM Identity Center** provides centralized identity management and enables **Single Sign-On (SSO)** across multiple AWS accounts and applications.

* Gained hands-on experience with **AWS Key Management Service (AWS KMS)** by:
  * Creating and managing encryption keys.
  * Understanding encryption at rest.
  * Applying AWS KMS to protect data stored across AWS services.
